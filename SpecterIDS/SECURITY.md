# Security Policy

## Purpose

SpecterIDS is defensive and educational. It is intended for authorized labs, training environments and portfolio review.

## Acceptable Use

Use this project only on systems and networks you own or have explicit permission to monitor. Follow local laws, institutional policy and responsible disclosure norms.

## What SpecterIDS Does Not Do

- It does not exploit vulnerabilities.
- It does not install persistence.
- It does not evade security tools.
- It does not steal credentials or payload data.
- It does not inject, modify or block network traffic.
- It does not perform automated attacks or retaliation.

## Limitations

Detection is metadata-based and heuristic. Alerts can be false positives or incomplete. SpecterIDS should support learning and investigation, not replace production monitoring, SIEM correlation or incident response procedures.

## Reporting Security Issues

If you find a bug that could cause unsafe behavior, memory corruption, data exposure or misleading security output, open a private security advisory if the project is hosted on GitHub, or contact the maintainer privately before public disclosure.

Please include:

- affected version or commit
- platform and compiler
- steps to reproduce
- expected and actual behavior
- any relevant logs or packet fixtures that are safe to share
